OpenFX : Documentation

Introduction

OFX documentation comes in three forms
  • Docbook based documentation, generated into two formats of HTML,
  • Doxygen html documentation generated from comments in header files,
  • Comments in the raw header files.
The source to the documentation, in the form of C header files, XML files, perl scripts and several other bits and pieces area available from the openfx CVS repository.

Base API Documentation

The base OFX API (as specified by the files in the project 'include' directory) is documented by a programmer's reference, a programming guide, doxygen documentation and header files.

Programing Reference

The OFX Programming Reference is mostly complete, and replaces the origin ungainly doxygen based reference pages. It is based on docbook and is available in three forms...
  • A browsable single large HTML file (754K),
  • A browsable set of HTML files, one per section,
  • A tarball of the set of HTML files (146K).

Programming Guide

The OFX Reference guide is very incomplete, it is new and is a guide on how to write plugins for the API. It is based on docbook and is available in three forms...
  • A single large HTML file (64K),
  • A set of HTML files, one per section,
  • A tarball of the set of HTML files (21K).

Doxygen Documentation

The doxygen documentation extracted from the header files is a useful adjunct to the programmming reference. It is available in two forms...
  • A set of HTML files, one per section,
  • A tarball of the set of HTML files (229K).

Header Files

The raw header files are available as a tarball.

C++ Support Library Documentation

The C++ skin ontop of the raw API is documented via doxygen based documentation and via the raw header files.

Doxygen Documentation

It is available in two forms...
  • A set of HTML files, one per section,
  • A tarball of the set of HTML files (229K).

Header Files

The raw header files are available from the CVS repository.
